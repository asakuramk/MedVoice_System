### 1.2 ステップ詳細

| ステップ | 処理内容 | 担当コンポーネント | 所要時間 |
|---------|---------|----------------|---------|
| 1 | 録音開始（AVAudioEngine + SFSpeechRecognizer 起動） | iOS アプリ | 即座 |
| 2 | リアルタイム文字起こし（録音中、UI に逐次表示） | SFSpeechRecognizer | 録音中 |
| 3 | 録音停止・最終テキスト確定（stopAndFinalize） | AppleSpeechTranscriber | 0 秒 |
| 4 | Chain 0: 医療用語誤認識修正（Claude） | バックエンド | 3 秒 |
| 5 | Chain 1: テキスト前処理・ノイズ除去（Claude） | バックエンド | 4 秒 |
| 6 | Chain 2: SOAP 形式レポート生成（Claude） | バックエンド | 10 秒 |
| 7 | Chain 3 + 4: サマリー・チェックリスト並列生成（Claude） | バックエンド | 8 秒 |
| 8 | Chain 5: 整合性検証（Claude）/ 不整合時は自動リトライ | バックエンド | 5 秒 |
| 9 | Google Docs 文書作成・フォーマット | Google Docs API | 5 秒 |
| 10 | Gmail 通知メール送信 | Gmail API | 3 秒 |
| **合計** | | | **約 38〜42 秒** |

### 2.3 バックグラウンド処理

```
録音終了
    │
    ├── アプリがフォアグラウンド？
    │       ├── Yes: 通常処理（API 呼び出し継続）
    │       └── No:  BGProcessingTask をスケジュール
    │                   │
    │                   └── iOS がバックグラウンドで API 処理を継続
    │                       完了時に Push 通知 + メール送信
    │
    └── 20 分後: Google Docs リンク付きメールが m@llll.jp に届く
```

## 3. エラーハンドリングフロー

### 3.1 音声認識エラー

```
SFSpeechRecognizer エラー発生
    │
    ├── エラーコード 203（タスク中断）
    │       └── 学会モードのチャンク切り替えによる正常動作 → 無視
    │
    ├── ネットワーク接続なし
    │       └── UI にエラーバナー表示「ネットワーク接続を確認してください」
    │           録音は継続（テキストが不完全になる可能性を通知）
    │
    └── 音声認識サービス利用不可
            └── UI にエラーバナー表示
                録音中断、再試行を促す
```

### 3.2 Chain 5 バリデーション失敗

```
Chain 5 検証結果: isValid = false
    │
    ├── issues に "critical" があるか？
    │       ├── Yes: attempt < maxRetries(2)?
    │       │           ├── Yes: Chain 2〜4 を再実行（修正指示付きプロンプト）
    │       │           └── No:  警告フラグ付きで結果を返す
    │       │                    医師に確認を促すバナーを文書に挿入
    │       │
    │       └── No（warning/info のみ）: そのまま処理続行
    │           Google Docs の修正ログセクションに warning を記録
    │
    └── 処理完了
```

### 3.3 Google Docs / Gmail エラー

```
Google Docs 作成失敗
    │
    ├── タイムアウト or 一時的エラー
    │       └── 5秒待機 → 1回リトライ
    │           └── 失敗: エラーログ、管理者アラート
    │               アプリ内に SOAP レポートをテキスト表示（フォールバック）
    │
    └── 認証エラー（Service Account 権限不足）
            └── 管理者アラート送信
                アプリ内にエラーメッセージ表示

Gmail 送信失敗
    │
    └── アプリ内通知で Google Docs URL を表示（メール不送信の代替）
        次回アプリ起動時に再送を試みる
```

## 6. セキュリティフロー

```
iOS アプリ起動
    │
    ├── JWT トークン有効期限チェック
    │       ├── 有効: そのまま使用
    │       └── 期限切れ: 自動更新リクエスト
    │
録音 → テキスト確定
    │
    ├── テキストはメモリ内のみ保持
    │   （ローカルストレージに保存しない）
    │
    ├── HTTPS (TLS 1.3) で API に送信
    │
    └── API 処理完了後
            ├── Claude API: テキストは処理後に即時破棄（保存なし）
            ├── Google Docs: 医師の Google Drive にのみ保存
            └── バックエンドサーバー: テキストの DB 永続化なし
```
