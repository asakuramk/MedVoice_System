## 1. システム目的と概要

MedVoice は、日本の個人クリニックおよび小規模医療機関の医師を対象とした、音声記録から医療文書を自動生成するシステムである。診察中の医師の発話をリアルタイムで文字起こしし、AI チェーン処理によって構造化された医療文書（SOAP 形式）を自動生成、Google Docs に保存してメールで通知する。

### 解決する課題

- 診察後の電子カルテ記入に要する時間（平均 15〜20 分/件）の削減
- 学会・勉強会の 60 分講演録音を構造化文書に自動変換
- 医療専門用語の音声認識誤りを AI が自動修正

### ターゲットユーザー

- 個人クリニック医師（診察モード）
- 小規模医療機関スタッフ（診察モード）
- 学会・勉強会参加医師（学会モード）

## 3. 動作モード比較

| 項目 | 診察モード | 学会モード |
|------|-----------|-----------|
| 録音時間 | 最大 10 分（典型 45 秒） | 最大 60 分 |
| 音声認識方式 | SFSpeechRecognizer（リアルタイム） | SFSpeechRecognizer（55 秒ループ） |
| AI 処理方式 | シーケンシャル 6 チェーン | 並列チャンク処理 → 統合 |
| 処理時間 | 合計 約 42 秒 | 合計 約 20 分 |

| コスト（Claude API） | 約 $0.33 / 件 | 約 $9.00 / 件 |
| バックグラウンド処理 | 不要 | BGProcessingTask 使用 |

## 5. データフロー（7 ステップ）

```
Step 1: 録音開始
  医師が iPhone の MedVoice アプリで録音ボタンをタップ
  → AVAudioEngine が PCM 16kHz モノラルでバッファリング開始
  → SFSpeechRecognizer がリアルタイム認識を開始

Step 2: リアルタイム文字起こし
  音声バッファが SFSpeechRecognizer に継続的に送られる
  → 部分的な認識結果が随時 UI に表示される
  → 学会モードの場合: 55 秒ごとにタスクを再起動してチャンク保存

Step 3: 録音終了・テキスト確定
  録音停止ボタンをタップ（または自動停止）
  → SFSpeechRecognizer が最終テキストを確定
  → 録音終了と同時に完全な文字起こしテキストが得られる
  → アプリがバックエンド API にテキストを POST

Step 4: Chain 0 — 医療用語修正
  バックエンドが受信テキストを Claude に送信
  → 音声認識による医療用語の誤り（例:「まくまれーてすと」→「McMurray test」）を修正
  → 修正ログを生成

Step 5: AI チェーン処理（Chain 1〜5）
  Chain 1: テキスト前処理・ノイズ除去・文章整形
  Chain 2: SOAP 形式構造化レポート生成
  Chain 3: 3 行実用サマリー生成
  Chain 4: 5 項目チェックリスト生成
  Chain 5: 整合性検証（不整合があれば Chain 2〜4 を自動リトライ）

Step 6: Google Docs 文書作成
  検証済みデータを Google Docs API で文書化
  → タイムスタンプ + タイトル形式でドキュメント作成
  → batchUpdate で書式設定（見出し・表・箇条書き）
  → 共有リンクを取得

Step 7: メール通知
  Gmail API で m@llll.jp に HTML メール送信
  → 文書へのリンク・サマリー・チェックリストを含む
  → 医師がメールから即座に確認・共有が可能
```

## 6. セキュリティ設計

### 6.1 データ保護

| 対策 | 内容 |
|------|------|
| 通信暗号化 | 全通信 TLS 1.3 |
| データ保存 | 音声データはサーバーに保存しない（テキスト変換後即時破棄） |
| テキストデータ | バックエンドメモリ内処理のみ、DB 永続化なし |
| Google Docs | 医師個人の Google アカウントに紐づく Service Account で管理 |
| JWT 認証 | 全 API エンドポイントに Bearer トークン必須 |
| トークン有効期限 | 1 時間（自動更新） |

### 6.2 アクセス制御

- iOS アプリ: マイク権限・音声認識権限を明示的に要求
- バックエンド API: JWT による認証必須
- Google Workspace: Service Account の権限を Docs/Gmail のみに限定
- Claude API キー: 環境変数管理、コードに埋め込み禁止

### 6.3 医療情報の取り扱い

- 患者氏名は音声認識テキストに含まれないよう運用ガイドで規定
- 生成文書は医師の Google Drive にのみ保存
- クラウドサービス（Claude API）に送信されるデータは匿名化推奨

## 8. コンプライアンス

### 8.1 個人情報保護法（改正個人情報保護法 2022 年施行）

- 医療情報は要配慮個人情報に該当
- 収集・利用目的の明示（診療記録作成目的）
- 第三者提供制限（Claude API 送信は業務委託として取り扱い）
- 本人からの開示・削除請求への対応手順を別途規定

### 8.2 医療法

- 電子カルテとして使用する場合は「真正性・見読性・保存性」の 3 原則に準拠
- 生成文書は医師が内容を確認・承認してから正式記録とする運用を推奨
- MedVoice 生成文書はドラフトとして位置づけ、最終確認は医師が行う

### 8.3 電子帳簿保存法

- Google Docs での保存は電子保存要件（訂正削除履歴管理）に対応
- Google Workspace の監査ログ機能を活用
