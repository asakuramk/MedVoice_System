## Phase 1: 事前準備

### 1-1. 必要なアカウント

| サービス | 用途 | URL |
|---------|------|-----|
| Anthropic Console | Claude API キー取得 | console.anthropic.com |
| Google Cloud Console | Docs API / Gmail API 有効化 | console.cloud.google.com |
| GitHub | コード管理 | github.com |
| Railway または Render | バックエンドホスティング | railway.app / render.com |
| Apple Developer Program | iOS アプリ配布（実機テスト用） | developer.apple.com |

### 1-2. Claude API キーの取得

```
1. console.anthropic.com にログイン
2. 「API Keys」→「Create Key」
3. キー名: medvoice-production
4. 生成されたキーを安全な場所に保存（再表示不可）
```

### 1-3. Google Cloud のセットアップ

```
1. console.cloud.google.com でプロジェクト作成
   プロジェクト名: medvoice-system

2. 以下の API を有効化
   - Google Docs API
   - Gmail API

3. Service Account を作成
   「IAM と管理」→「サービス アカウント」→「作成」
   名前: medvoice-backend
   ロール: 編集者

4. JSON キーを発行・ダウンロード
   サービス アカウント → 「キー」タブ → 「鍵を追加」→ JSON

5. 通知先メールアドレス（m@llll.jp）に
   Service Account のメールアドレスからアクセス許可が必要なため、
   Google Drive の共有設定で当該アカウントにアクセス権を付与する
```

### 1-4. ローカル開発環境の準備

```bash
# Node.js（バックエンド）
node --version   # 20.x LTS 以上を確認

# Xcode（iOS）
# Mac App Store から最新版をインストール
xcode-select --install

# パッケージマネージャー
npm install -g typescript ts-node
```

## Phase 3: iOS アプリ構築

### 3-1. Xcode プロジェクトの作成

```
1. Xcode を起動
2. 「Create New Project」
3. iOS → App を選択
   Product Name: MedVoice
   Bundle Identifier: jp.medvoice.app
   Interface: SwiftUI
   Language: Swift
   Use Core Data: OFF
```

### 3-2. Info.plist への権限追加

Xcode のターゲット設定 → Info タブで以下を追加:

```
NSMicrophoneUsageDescription
  → "診察・学会の音声を録音して医療文書を自動生成するために使用します"

NSSpeechRecognitionUsageDescription
  → "録音音声をリアルタイムでテキストに変換するために使用します"

BGTaskSchedulerPermittedIdentifiers
  → jp.medvoice.app.academic-processing
```

### 3-3. 実装順序

```
Step 1: AudioRecorder.swift
        録音の開始・停止が動作することを実機で確認

Step 2: AppleSpeechTranscriber.swift
        録音中にリアルタイムでテキストが表示されることを確認
        （02_iOS_アプリ仕様書.md のコードをそのまま使用）

Step 3: RecordingView.swift + RecordingViewModel.swift
        UI から録音・停止ができることを確認

Step 4: APIClient.swift
        バックエンドにテキストを POST できることを確認

Step 5: ResultView.swift
        Google Docs URL とサマリーが表示されることを確認

Step 6: 学会モードの 55 秒チャンクループを追加
        60 分録音でテキストが途切れずに蓄積されることを確認
```

### 3-4. バックエンドの URL 設定

```swift
// APIClient.swift
#if DEBUG
let baseURL = "http://localhost:3000"  // ローカル開発時
#else
let baseURL = "https://your-app.railway.app"  // 本番環境

```

### 3-5. 実機での動作確認チェックリスト

```
[ ] マイク権限ダイアログが表示される
[ ] 音声認識権限ダイアログが表示される
[ ] 録音開始と同時にテキストが表示され始める
[ ] 録音停止と同時にテキストが確定する（待ち時間なし）

[ ] m@llll.jp にメールが届く
```

## Phase 4: Google Docs / Gmail の最終設定

### 4-1. Google Docs の共有設定

バックエンドが作成したドキュメントを m@llll.jp が受信・閲覧できるよう、
Service Account から自動共有する設定を `docsService.ts` に追加する:

```typescript
// ドキュメント作成後、m@llll.jp に閲覧権限を付与
await drive.permissions.create({
  fileId: documentId,
  requestBody: {
    type: 'user',
    role: 'writer',
    emailAddress: process.env.NOTIFY_EMAIL,
  },
});
```

### 4-2. メール送信テスト

```bash
ts-node src/test/testFullFlow.ts
```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| 音声認識が開始されない | 権限未付与 | 設定 → プライバシー → 音声認識をオン |
| API エラー 401 | JWT トークン不正 | `.env` の JWT_SECRET を確認 |
| Google Docs が作成されない | Service Account キー誤り | JSON キーの改行を `\n` に置換して `.env` に設定 |
| メールが届かない | Gmail API 未有効化 | Google Cloud Console で Gmail API を有効化 |
| Chain 0 で補正されない | プロンプト設計の問題 | `testChain0.ts` で単体テストして出力を確認 |
| 学会モードでテキストが途切れる | チャンクタイマーの不具合 | `AppleSpeechTranscriber` の `rotateChunk()` をデバッグ |
