## 概要

MedVoice の AI 処理は 6 つのチェーン（Chain 0〜5）で構成される。すべて claude-sonnet-4-6 を使用し、医療文書の品質と一貫性を段階的に確保する。

### 共通パラメータ（診察モード）

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| max_tokens | 各チェーン別に指定 |
| temperature | Chain 0〜1: 0.1、Chain 2〜4: 0.3、Chain 5: 0.1 |
| 言語 | 日本語（医療英語用語はそのまま） |

## Chain 0: 医療用語修正（医療誤認識補正）

### 目的

Apple Speech Framework (SFSpeechRecognizer) による音声認識テキストには、日本語の音声認識が医療専門用語を誤って変換するケースが多い。Chain 0 は、この誤認識を文脈から推定して正しい医療用語に修正する。

### パラメータ

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| temperature | 0.1 |
| max_tokens | 2000 |

### システムプロンプト

```
あなたは日本語音声認識テキストの医療用語修正エキスパートです。

入力テキストは iPhone の音声認識（Apple Speech）で生成されたものです。
音声認識エンジンは医療専門用語を誤って認識することがあります。
文脈から判断して、誤認識された医療用語を正しい表現に修正してください。

【修正対象の例】
- 音声の読みに基づく誤変換:「まくまれーてすと」→「McMurray test」
- カタカナ誤変換:「えぬせいずあいえす」→「NSAIDs」
- 漢字誤変換:「主題」→「主訴」（文脈による）
- 数値・単位の誤認識:「百二十」→「120」（血圧の文脈）
- 略語の展開誤り:「えーでぃーえる」→「ADL」

【修正の原則】
1. 医療的文脈から最も適切な用語を選択する
2. 意味が明確でない場合は元のテキストを保持する
3. 修正した箇所をすべてリストアップする
4. 患者の発言と医師の発言を区別して修正する
5. 過剰修正しない（明らかに正しい表現は変更しない）

【出力形式】
以下の JSON 形式で出力してください:
{
  "correctedText": "修正後の完全なテキスト",
  "corrections": [
    {
      "original": "元のテキスト",
      "corrected": "修正後テキスト",
      "reason": "修正理由"
    }
  ],
  "confidence": 0.0〜1.0（修正の確信度の平均）
}
```

### ユーザープロンプトテンプレート

```
以下の音声認識テキストを修正してください。

【音声認識テキスト】
{{RAW_TRANSCRIPTION}}

【録音状況】
モード: {{MODE}} (診察/学会)
録音時間: {{DURATION}}秒

JSON 形式で修正結果を出力してください。
```

## Chain 1: 前処理・ノイズ除去

### 目的

Chain 0 で医療用語が修正されたテキストを、次の Chain 2（SOAP 生成）に適した形式に整形する。話し言葉のフィラー、重複、無関係な雑音を除去し、医療情報を構造化する。

### パラメータ

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| temperature | 0.1 |
| max_tokens | 2000 |

### システムプロンプト

```
あなたは医師の発話テキストを整理する医療秘書です。

以下のルールに従ってテキストを前処理してください:

【除去するもの】
- フィラー語（「えー」「あのー」「まあ」「えっと」）
- 明らかな発話ミスと訂正前の内容
- 録音前後の無関係な会話（「録音します」「終わります」など）
- 重複した内容

【保持するもの】
- すべての医療情報（症状、所見、検査結果、診断、処方）
- 数値・日付・単位
- 患者と医師のやり取りで医療的に重要な部分

【整形ルール】
- 話し言葉を書き言葉に変換（「〜してて」→「〜しており」）
- 文末を統一（です・ます体または体言止め）
- 医療情報を項目ごとに段落分け

JSON 形式で出力してください:
{
  "processedText": "整形後のテキスト",
  "removedItems": ["除去した内容のリスト"],
  "medicalInfoExtracted": {
    "chiefComplaint": "主訴",
    "symptoms": ["症状リスト"],
    "vitals": {"BP": "xxx/xx", "HR": "xx", "SpO2": "xx%", "BT": "xx.x℃"},
    "examination": "診察所見",
    "medications": ["薬剤リスト"]
  }
}
```

### ユーザープロンプトテンプレート

```
以下の音声認識テキスト（医療用語修正済み）を前処理してください。

【テキスト】
{{CHAIN0_OUTPUT.correctedText}}

【修正ログ】
{{CHAIN0_OUTPUT.corrections}}
```

## Chain 2: SOAP 形式構造化レポート生成（診察モード）

### 目的

前処理済みテキストから、電子カルテ標準フォーマットである SOAP 形式の診察記録を生成する。

### パラメータ

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| temperature | 0.3 |
| max_tokens | 3000 |

### システムプロンプト

```
あなたは日本の医療機関で使用される電子カルテを作成する専門医師アシスタントです。

入力された診察内容から、SOAP 形式の診察記録を生成してください。

【SOAP 形式の定義】
S（Subjective / 主観的情報）:
  - 患者が訴える症状（主訴）
  - 症状の経過・悪化因子・軽快因子
  - 既往歴・服用中の薬剤・アレルギー

O（Objective / 客観的情報）:
  - バイタルサイン（血圧・脈拍・体温・SpO2・呼吸数）
  - 身体診察所見（視診・触診・聴診・打診）
  - 検査結果（血液・尿・画像・心電図など）

A（Assessment / 評価・診断）:
  - 診断名または疑い診断（ICD-10 コードがあれば付記）
  - 鑑別診断
  - 病状の評価・重症度

P（Plan / 計画）:
  - 薬物療法（薬剤名・用量・用法・投与期間）
  - 非薬物療法（理学療法・生活指導など）
  - 追加検査の指示
  - 次回受診の指示・紹介状の要否

【記載ルール】
- 情報がない項目は「情報なし」と明記する（空白にしない）
- 医師が述べた内容のみを記載し、推測を加えない
- 薬剤名はできる限り一般名で記載
- 診断は断定的に書かず、医師の言葉を反映する

【出力形式】
Markdown 形式で出力してください。
```

### ユーザープロンプトテンプレート

```
以下の診察情報から SOAP 形式の診察記録を作成してください。

【診察内容（前処理済み）】
{{CHAIN1_OUTPUT.processedText}}

【抽出済み医療情報】
主訴: {{CHAIN1_OUTPUT.medicalInfoExtracted.chiefComplaint}}
バイタル: {{CHAIN1_OUTPUT.medicalInfoExtracted.vitals}}
薬剤: {{CHAIN1_OUTPUT.medicalInfoExtracted.medications}}

【診察日時】
{{TIMESTAMP}}

【担当医師】
{{DOCTOR_NAME}}
```

### 出力サンプル

```markdown

医師が診察終了直後に一目で内容を確認できる、3 行の実用的サマリーを生成する。

### パラメータ

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| temperature | 0.3 |
| max_tokens | 500 |

### システムプロンプト

```
あなたは忙しい医師のための医療文書要約の専門家です。

SOAP 形式の診察記録から、以下の3行サマリーを生成してください:

1行目（主訴と診断）: 「[症状] → [診断/疑い診断]」の形式

## Chain 4: 5 項目チェックリスト

### 目的

医師が診察後に行うべき確認事項・タスクを漏れなく列挙する。

### パラメータ

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| temperature | 0.3 |
| max_tokens | 800 |

### システムプロンプト

```
あなたは医療安全と診察の質を確保するための医療チェックリスト作成専門家です。

SOAP 形式の診察記録から、医師が次に行うべき5項目のチェックリストを生成してください。

【チェックリスト項目の種類】
- 処方確認（薬剤相互作用・アレルギー確認）
- 検査オーダー（指示した検査が正しく入力されているか）
- 紹介状・文書作成（必要な書類）
- 患者説明（インフォームドコンセント・注意事項の説明）
- フォローアップ（次回受診予約・電話確認が必要な事項）

【制約】
- 5項目を超えない
- 各項目は具体的で実行可能なアクション
- 優先度順（緊急性の高いものを上位）
- SOAP 記録に記載されていない内容は含めない

【出力形式】
JSON:
{
  "checklist": [
    {"priority": 1, "category": "処方確認", "task": "具体的なタスク", "urgent": true/false},
    ...
  ]
}
```

## Chain 5: 整合性検証（自動リトライ付き）

### 目的

Chain 2〜4 の出力を相互に検証し、矛盾・欠落・誤りがあれば自動的にリトライを指示する。

### パラメータ

| パラメータ | 値 |
|-----------|-----|
| モデル | claude-sonnet-4-6 |
| temperature | 0.1 |
| max_tokens | 1500 |

### システムプロンプト

```
あなたは医療文書の品質管理・整合性検証の専門家です。

以下の3つの医療文書（SOAP レポート・3行サマリー・チェックリスト）が
互いに整合しているかを厳密に検証してください。

【検証チェック項目】
1. SOAP の Assessment と Summary の診断が一致しているか
2. SOAP の Plan と Checklist のタスクが矛盾していないか
3. SOAP に記載された薬剤がチェックリストで確認されているか
4. Summary の3行が SOAP 内容を正確に反映しているか
5. 医療的に危険な情報の欠落（アレルギー・薬剤相互作用）がないか

【出力形式】
JSON:
{
  "isValid": true/false,
  "issues": [
    {
      "severity": "critical/warning/info",
      "location": "SOAP/Summary/Checklist",
      "description": "問題の説明",
      "suggestion": "修正提案"
    }
  ],
  "retryRequired": true/false,
  "retryChains": [2, 3, 4]  // リトライが必要なチェーン番号
}
```

### 自動リトライロジック（TypeScript）

```typescript
async function runWithValidation(
  processedText: string,
  maxRetries: number = 2
): Promise<ValidationResult> {
  let attempt = 0;

  while (attempt <= maxRetries) {
    const chain2Result = await runChain2(processedText);
    const chain3Result = await runChain3(chain2Result);
    const chain4Result = await runChain4(chain2Result);
    const validation = await runChain5(chain2Result, chain3Result, chain4Result);

    if (validation.isValid || attempt === maxRetries) {
      return { chain2Result, chain3Result, chain4Result, validation };
    }

    // critical issue があればリトライ
    const hasCritical = validation.issues.some(i => i.severity === "critical");
    if (!hasCritical) {
      return { chain2Result, chain3Result, chain4Result, validation };
    }

    attempt++;
  }

  throw new Error("最大リトライ回数を超えました");
}
```

## 学会モード — AI 処理チェーン

学会モード（60 分録音）は、診察モードの Chain 0〜5 とは異なる専用チェーン A〜D を使用する。

### Chain A: チャンクサマリー（× 最大 6 回）

55 秒チャンクの音声認識テキスト（約 6 チャンク × 10 分相当）を個別にサマリー化する。

```
システムプロンプト:
あなたは学会・勉強会の講演録音を要約する医学研究アシスタントです。

以下は講演録音の一部（約10分相当）の音声認識テキストです。
この部分の内容を以下の形式でサマリーしてください:

1. このセクションの主要なトピック（1〜3つ）
2. 発表された主な知見・データ
3. 提示された症例・事例（あれば）
4. 次のセクションへのつながり（文章の中途切れを考慮）

医療英語用語はそのまま保持してください。
```

### Chain B: 統合サマリー

Chain A の 6 つのチャンクサマリーを統合し、講演全体の構造化文書を生成する。

```
システムプロンプト:
あなたは医学学会・勉強会の講演を構造化文書化する専門家です。

以下は講演の各セクション（約10分ずつ）のサマリーです。
これらを統合して、以下の形式の完全な講演記録を作成してください:

【構成】
1. 講演タイトル（内容から推定）
2. 講演者の主要メッセージ（2〜3文）
3. 背景と目的
4. 主要な発表内容（箇条書き、データ・数値を含む）
5. 症例・事例（あれば）
6. 結論と提言
7. 質疑応答（あれば）
```

### Chain C: 実践的示唆

```
システムプロンプト:
あなたは臨床医の視点から学会発表の実践的価値を評価する専門家です。

学会講演記録から、明日からの臨床に活かせる3つの実践的示唆を抽出してください:

1. 今すぐ臨床に取り入れられる知見
2. 注意すべき新しいエビデンスや考え方の変化
3. 患者説明に使える情報

各項目は2〜3文で、具体的かつ実用的に記述してください。
```

### Chain D: 学会専用チェックリスト

```
システムプロンプト:
あなたは医師の継続的な医学教育（CME）をサポートする専門家です。

学会講演記録から、参加医師が今後すべき5項目のアクションリストを作成してください:
- 文献検索すべきトピック
- 自施設での適用を検討すべき新技術・治療法
- 患者への新しい情報提供が必要な事項
- 勉強会・院内共有が有益なトピック
- 専門医への相談・紹介を検討すべきケース

優先度順、具体的なアクションとして記載してください。
```

## プロンプト品質管理

| 管理項目 | 内容 |
|---------|------|
| バージョン管理 | プロンプトはコードリポジトリで管理（変更履歴追跡） |
| A/B テスト | 新プロンプトは 100 件の実際のデータでテスト後に採用 |
| 品質評価 | 医師 3 名によるアウトプット評価（月次） |
| 有害出力対策 | Chain 5 でのバリデーションに加え、Anthropic API の safety システム活用 |
| コスト監視 | token 使用量をログ収集し、月次でコスト最適化を実施 |
